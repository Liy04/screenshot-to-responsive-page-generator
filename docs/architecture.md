# 架构说明

本文档描述项目未来的整体架构。当前 Week 01 不实现这些模块，只用于明确后续方向。

## 整体关系

```text
用户浏览器
  |
  v
frontend 前端应用
  |
  v
backend 后端 API 服务
  |
  +--> MySQL 数据库
  |
  +--> worker Python 异步处理
          |
          v
       生成页面结果
```

## frontend 前端

`frontend/` 后续用于存放 Vue3 + Vite 前端工程。

主要职责：

- 提供截图上传界面。
- 展示生成任务列表和任务状态。
- 展示 mock 生成结果。
- 后续展示真实响应式页面预览。

## backend 后端

`backend/` 后续用于存放 Java + Spring Boot 后端工程。

主要职责：

- 提供 RESTful API（面向资源的接口）。
- 接收截图上传请求。
- 创建和查询生成任务。
- 保存任务记录和生成结果。
- 调度 Python Worker。

## worker Python 处理器

`worker/` 后续用于存放 Python Worker。

主要职责：

- 接收后端分发的生成任务。
- 解析截图或设计输入。
- 后续调用 AI 模型生成页面代码。
- 返回生成结果和任务状态。

## MySQL 数据库

MySQL 用于保存持久化数据。

后续可能保存：

- 任务信息。
- 上传文件信息。
- 生成结果。
- 任务状态变化记录。

Week 01 不设计数据库表。

## 后续 Redis

Redis 是内存数据存储，后续可能用于：

- 缓存任务状态。
- 保存短期会话数据。
- 做简单的限流或临时队列。

当前阶段不引入 Redis。

## 后续 RabbitMQ

RabbitMQ 是消息队列，后续可能用于：

- 后端向 Worker 投递异步任务。
- 解耦接口请求和生成处理。
- 支持任务重试和失败处理。

当前阶段不引入 RabbitMQ。

## 未来请求流程

```text
1. 用户在前端上传截图。
2. 前端调用后端上传接口。
3. 后端保存文件并创建任务。
4. 后端将任务交给 Worker。
5. Worker 生成页面结果。
6. 后端保存结果。
7. 前端查询任务状态并展示结果。
```

# 测试说明

本文档说明后续项目的测试方式。当前 Week 01 只建立测试策略，不编写测试脚本。

## 测试目标

测试目标是保证项目最小闭环稳定可用。

MVP 第一版阶段重点验证：

- 前端页面能打开。
- 截图可以上传。
- 后端可以创建任务。
- 任务状态可以查询。
- mock 生成结果可以展示。

## 接口测试

工具：Postman（接口测试工具）。

后续需要测试：

- 上传截图接口。
- 创建任务接口。
- 查询任务详情接口。
- 查询任务列表接口。
- 获取生成结果接口。

接口测试重点：

- 请求方法是否正确。
- 参数名是否正确。
- 返回结构是否稳定。
- 异常情况是否有明确提示。

## 冒烟测试

冒烟测试是 Smoke Test（快速基础验证），用于确认核心功能没有明显损坏。

后续每次完成阶段功能后，至少验证：

- 前端能正常启动。
- 后端能正常启动。
- 首页能访问。
- 上传入口可见。
- 基础接口能返回结果。

## 前端页面测试

后续可以使用 Web 自动化测试验证页面。

重点检查：

- 页面在桌面端和移动端是否可用。
- 上传按钮、任务列表、状态展示是否正常。
- 生成结果区域是否能展示内容。
- 错误提示是否清晰。

## 最小闭环测试

最小闭环测试用于验证 MVP 主流程：

```text
打开前端页面 -> 上传截图 -> 创建任务 -> 查询状态 -> 查看 mock 结果
```

只要这个流程能稳定走通，就说明 MVP 的核心路径可用。

## 性能测试

工具：JMeter（性能测试工具）。

早期不做复杂性能测试。后续接口稳定后，再考虑：

- 并发上传。
- 并发查询任务状态。
- Worker 处理耗时。
- 后端响应时间。

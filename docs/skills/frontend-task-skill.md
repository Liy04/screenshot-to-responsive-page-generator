# Frontend Task Skill Draft

## 文件目的

本文件用于沉淀前端任务的固定协作套路，减少重复解释。

## 适用场景

以下任务适合使用本 skill：

- 新增一个页面
- 新增一个基础组件
- 根据截图做页面占位实现
- 调整响应式布局
- 修复视觉层级、间距、对齐问题

## 当前 MVP 阶段边界

当前阶段前端任务以 `docs/context/current-phase.md` 和当前任务卡为准。Week 02 优先服务创建任务页、任务详情页和前后端最小闭环。

允许做：

- Vue3 + Vite 最小页面和组件。
- 页面标题、阶段说明、基础状态展示。
- 简单响应式布局。
- 与 mock API 对接前的请求封装准备。

暂不做：

- 不接入真实模型生成能力。
- 不接入 Figma API / Figma MCP。
- 不实现真实截图解析。
- 不实现真实代码生成。
- 不实现拖拽编辑器、多页面编辑器或导出 zip。
- 不引入大型 UI 组件库，除非用户明确确认。
- 不提交 `node_modules/`、`dist/`、临时日志文件或其它构建产物。

## 标准步骤

1. 先阅读 `AGENTS.md`
2. 再阅读 `docs/context/current-phase.md`、当前任务卡、必要专项文档和相关页面代码
3. 先输出计划，等待确认后再执行
4. 优先复用现有组件和样式写法
5. 改动尽量小，不做无关重构
6. 改完后优先运行 `npm run build`
7. 需要页面检查时，短暂运行 `npm run dev -- --host 127.0.0.1 --port 5173`，验证完成后停止进程
8. 输出改动文件、改动说明、验证结果和风险说明

## 产出要求

前端任务完成后，至少要汇报：

- 改动了哪些文件
- 新增了哪些组件或页面
- 使用了哪些已有模式
- 如何启动查看
- 如何验证结果
- 是否触及当前 MVP 边界

## 验收标准

满足以下条件时，视为本 skill 有效：

- 使用者能直接复用这套步骤发起前端任务
- Codex 能按这套步骤先计划再改动
- 结果汇报格式一致，便于复盘

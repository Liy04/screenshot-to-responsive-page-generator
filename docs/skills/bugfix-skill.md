# Bug 修复任务工作流

本工作流用于指导 Codex 修复问题。

Bug 是软件缺陷，修复时要先定位根因，再做最小修改。

## 适用场景

- 页面显示异常。
- 接口返回错误。
- 上传失败。
- 状态展示不正确。
- 构建、编译或测试失败。

## 执行步骤

1. 阅读 `AGENTS.md`、`README.md`、`docs/week/01-plan.md`、`docs/week/01-status.md` 和相关模块文档。
2. 复现问题，记录实际现象。
3. 定位根因，区分前端、后端、Worker 或配置问题。
4. 输出修复计划，说明影响范围和准备修改的文件。
5. 等待确认后执行。
6. 做最小必要修改。
7. 运行能覆盖问题的验证。
8. 汇报根因、修复内容、验证结果和剩余风险。

## 修复原则

- 优先修复根因，不只掩盖现象。
- 不做无关重构。
- 不扩大修改范围。
- 不删除用户已有改动。
- 对临时方案必须说明原因和后续替换方向。
- 测试线程只报告问题、复现步骤和影响范围，不直接修改业务代码。
- 修复线程负责最小修改、回归验证和风险说明。
- 如果问题来自文档与真实命令不一致，优先同步文档，不改动已验证可用命令。
- 如果修复会突破 Week 01 禁止事项，必须先说明原因并等待确认。

## 验证建议

根据问题类型选择验证方式：

- 前端问题：优先运行 `npm run build`；如果项目已配置 lint 脚本，再运行 `npm run lint`；必要时手动检查页面。
- 后端问题：运行 `mvn test`、`mvn package -DskipTests`，必要时用 `java -jar target/backend-0.0.1-SNAPSHOT.jar` 启动并验证接口。
- Worker 问题：运行对应 Python 脚本或单元测试。
- 跨模块问题：执行最小闭环测试。

# MVP 第一版范围

MVP 是 Minimum Viable Product（最小可行产品），指第一版只实现最小闭环，先证明项目流程可走通，再逐步增强。

## 第一版目标

第一版目标是完成“截图输入到 mock 生成结果展示”的最小流程。

用户能够：

- 上传一张页面截图。
- 创建一个生成任务。
- 查看任务状态。
- 查看一个 mock（模拟的）生成结果。

## 第一版包含内容

### 截图上传

支持用户上传一张截图文件，作为页面生成的输入。

第一版只需要支持基础上传流程，不要求复杂图片处理。

### 任务创建

上传截图后，后端创建一个生成任务，记录任务的基础信息和状态。

### Mock 生成结果

第一版不接入真实 AI 生成，只返回模拟结果。

Mock 结果可以是固定 HTML、CSS 或简单页面预览数据，用来验证前后端流程。

### 状态展示

前端展示任务状态，例如：

- 待处理。
- 处理中。
- 已完成。
- 失败。

## 第一版暂不包含内容

- 不做 Figma 集成。
- 不接入真实 AI 模型 API。
- 不做真实响应式页面生成。
- 不做拖拽编辑器。
- 不做多页面生成。
- 不做用户登录和权限系统。
- 不做复杂文件管理。
- 不做数据库高级设计。
- 不做任务队列和分布式处理。

## 第一版成功标准

只要能完成以下流程，就认为 MVP 第一版成功：

```text
上传截图 -> 创建任务 -> 查看任务状态 -> 查看 mock 生成结果
```
